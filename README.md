# AI 라이프 코치.

캘린더 일정, 운동/식단/체중 기록, 학업 진도, 그리고 그날그날의 맥락(피로도·고민·질문)을
채팅으로 입력하면 Supabase(PostgreSQL + pgvector)에 저장하고, 하이브리드 RAG(정형 SQL 통계 +
벡터 유사도 검색)로 과거 기록을 대조해 개인화된 조언을 주는 단일 파일(`index.html`) 웹앱입니다.

## 구성

- `supabase/schema.sql` — Supabase에 실행할 DB 스키마 (테이블 + pgvector + RPC 함수 + RLS)
- `index.html` — 앱 전체 (Tailwind CSS + Supabase JS + Chart.js, 전부 CDN, 빌드 불필요)

## 설정 방법

### 1. Supabase 프로젝트 준비
1. [supabase.com](https://supabase.com)에서 새 프로젝트 생성
2. 프로젝트 대시보드 → **SQL Editor** → `supabase/schema.sql` 내용 전체 붙여넣기 → 실행
3. 프로젝트 대시보드 → **Settings → API**에서 `Project URL`과 `anon public` key 확인

### 2. Gemini API Key 준비
- [Google AI Studio](https://aistudio.google.com/apikey)에서 Gemini API Key 발급
- 이 앱은 채팅 응답과 임베딩(벡터 검색) 모두 **Gemini API 하나로 통일**되어 있습니다.
  - 채팅: `generateContent` (모델명은 설정 화면에서 직접 입력, 예: `gemini-2.5-flash`)
  - 임베딩: `text-embedding-004` (768차원 — `schema.sql`의 `vector(768)` 컬럼과 반드시 일치해야 합니다)
- 프로덕션처럼 여러 사람이 접근할 수 있는 환경이라면 Google Cloud 콘솔에서 API 키에 HTTP 리퍼러 제한을 걸어두는 것을 권장합니다.

### 3. 앱 배포 (GitHub Pages)
1. 이 저장소를 GitHub Pages로 배포 (Settings → Pages → 브랜치 선택 → 루트 `/`)
2. 배포된 URL로 접속 → 스마트폰에서 "홈 화면에 추가"로 독립 앱처럼 사용 가능
3. 로컬 테스트만 할 경우 `index.html`을 브라우저에서 더블클릭해서 바로 열어도 동작합니다.

### 4. 앱 내 설정
1. 우측 상단 ⚙️ 버튼 클릭
2. Supabase URL / anon key 입력
3. 채팅 모델명 입력 (예: `gemini-2.5-flash`)
4. Gemini API Key 입력
5. "연결 테스트" → 성공 확인 후 "저장"

모든 키는 **브라우저 localStorage**에만 저장되며, 이 앱 자체는 외부 서버 없이 브라우저에서
Supabase/Gemini로 직접 API를 호출하는 구조입니다. anon key와 API key는 사실상
내 DB 전체 권한 및 결제 권한을 가지므로 타인과 공유하거나 공개 저장소 커밋 메시지 등에 남기지 마세요.

## 핵심 동작

- **캘린더가 홈 화면**: 앱을 열면 월간 캘린더 그리드가 바로 보입니다. 날짜를 탭하면 그 날의
  확정 일정과 자유 타임라인/할 일 메모를 입력할 수 있고, 일정이 있는 날에는 점(dot)으로 표시됩니다.
- **마감 기한 기반 "할 일"**: 캘린더 아래 별도 섹션에서 날짜에 묶이지 않고 "언제까지 해야 한다"만
  입력하면 됩니다. 각 할 일의 "🤖 AI에게 언제 할지 추천받기" 버튼을 누르면, 오늘부터 마감일까지의
  확정 일정 사이 빈 시간을 분석해 구체적인 시간대와 방식(예: "화요일 저녁 약속과 수요일 회의 사이,
  30분 홈트")을 제안하고 결과가 저장됩니다. 채팅으로 "다음 주 금요일까지 과제 있어"처럼 말해도
  자동으로 할 일로 등록됩니다.
- **목표는 학업/운동을 기본 카테고리로, 무제한 추가 가능**: 목표 탭은 🎓 학업 목표 / 💪 운동 목표
  / 📌 기타 목표 세 그룹으로 나뉘어 있고, 각 그룹에 여러 개를 등록할 수 있습니다. "+ 다른 목표
  추가" 버튼으로 언제든 새 목표를 만들 수 있습니다. 목표 기간은 정확한 날짜 대신 "5월 중순까지",
  "이번 달 말까지" 같은 자연어로 입력하면 AI가 오늘 날짜 기준으로 알아서 해석합니다.
- **실시간 동기화**: 모든 데이터는 Supabase 원격 DB에 즉시 저장되고, `postgres_changes`
  Realtime 구독을 통해 다른 탭/기기에서도 자동 반영됩니다 (`schema.sql` 마지막 블록이
  각 테이블을 `supabase_realtime` publication에 등록합니다).
- **선제적 일일 브리핑**: "코치" 탭을 열었을 때 오늘자 코치 메시지가 아직 없으면, 사용자가
  아무것도 입력하지 않아도 AI가 먼저 오늘 일정·목표·최근 기록을 분석해 시간대별 가이드를
  제시합니다. 상단의 "오늘의 브리핑 다시 받기" 버튼으로 언제든 재요청할 수도 있습니다.
- **매크로 대조 분석**: 최근 7일 단백질/탄수화물/지방 합계를 운동 볼륨·체중 추이와 함께
  코칭 컨텍스트에 포함해 "탄수화물 부족 → 오늘 중량 저하 위험" 같은 정밀 피드백이 가능합니다.

## 설정값이 계속 저장되게 하기

설정(⚙️)에서 입력한 Supabase URL/key와 Gemini API Key는 브라우저 `localStorage`에 자동
저장되어, 같은 브라우저로 **같은 주소(URL)** 에 다시 접속하면 다시 입력할 필요가 없습니다.
다만 아래 경우에는 저장이 유지되지 않을 수 있습니다.

- `index.html` 파일을 매번 새로 다운로드해서 로컬로 여는 경우 (특히 모바일에서 파일 미리보기로
  열면 저장소가 아예 없는 임시 뷰어로 열릴 수 있습니다) → **GitHub Pages 같은 고정 URL로
  배포해서 그 주소로만 접속**하는 것이 근본적인 해결책입니다.
- 시크릿/프라이빗 브라우징 모드로 접속한 경우
- 브라우저의 사이트 데이터를 수동으로 삭제한 경우

이런 상황을 대비해 설정 화면 하단에 **"설정 내보내기(복사)" / "설정 가져오기(붙여넣기)"**
버튼을 추가해뒀습니다. 한 번 내보내기로 백업 코드를 복사해 메모장 등에 보관해두면, 저장이
초기화되더라도 붙여넣기 한 번으로 즉시 복원할 수 있습니다.

## 알려진 제약사항 (설계상 트레이드오프)

- **구글/삼성 캘린더 실시간 연동(OAuth)** 은 포함하지 않았습니다. 정적 단일 HTML 파일 구조에서
  OAuth 클라이언트 시크릿을 안전하게 다루려면 별도 백엔드가 필요하기 때문입니다. 대신
  캘린더 탭에서 날짜를 선택해 일정/타임라인을 직접 입력하거나 캘린더 앱 내용을 복사해
  붙여넣는 방식으로 동일한 효과(확정 일정을 AI가 맥락으로 활용)를 구현했습니다.
- 채팅에 자연어로 기록을 입력하면 LLM이 JSON으로 구조를 추출해 SQL 테이블에 나누어 저장합니다.
  드물게 LLM이 형식을 벗어난 응답을 하면 자동으로 "구조화 실패 → 전체 텍스트를 맥락 기억(벡터)으로만
  저장"하는 폴백이 동작합니다. 이 경우 "기록" 탭의 수동 입력창을 사용하면 100% 정확하게 저장됩니다.
- API 키가 브라우저에 그대로 노출되는 구조이므로, 개인 전용 사용을 전제로 합니다. 여러 사람과
  같이 쓰는 용도로는 적합하지 않습니다.
- (이전에 OpenAI 임베딩 기준 `vector(1536)`으로 스키마를 이미 실행한 적이 있다면) 지금 버전은
  Gemini `text-embedding-004` 기준 `vector(768)`로 바뀌었으므로, Supabase SQL Editor에서
  `context_memories` 테이블을 `drop table context_memories;` 로 지우고 `schema.sql`을 다시
  실행해야 합니다 (기존 벡터 데이터는 차원이 달라 재사용할 수 없습니다).
